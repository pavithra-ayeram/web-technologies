Lab 3: Event Registration Form

1. Question

Develop a registration form that validates user input for a community event signup using HTML, CSS, and JavaScript.

  • Requirement Analysis: Define all form fields and validation requirements (e.g., name, email, phone number, age). Specify which fields are required, valid formats (like regex for email), and any custom rules (e.g., age must be over 18).
  • Design: Sketch a user-friendly registration form UI that provides feedback for invalid fields (e.g., red outlines, error messages). Use a wireframing tool, such as Figma, or create a hand-drawn sketch. Make sure the design is modular.
  • Implementation: Build the form using HTML for structure, CSS for styling, and JavaScript for real-time form validation.
  • Testing: Test form validation for all scenarios: required fields missing, invalid formats, boundary values (e.g., age 17 vs 18). Document edge cases and ensure clear error handling.
  • Deployment: Host your project on GitHub with a structured repository, including a README explaining form validation rules. Deploy the form live using GitHub Pages or a similar free service.
